На чипе ESP32-C6 доступны **три** аппаратных последовательных порта, которые 
можно использовать в ваших проектах. В зависимости от конкретной платы
разработчика, некоторые из этих портов могут быть выведены на USB-разъем или
доступны на контактных площадках (пинах) GPIO.

Вот их подробное описание:

### 1. USB Serial/JTAG Controller (USB-порт)
Это уникальная для ESP32-C6 возможность. Это не просто UART, а полноценный USB-
контроллер, встроенный в чип. Он позволяет подключаться к компьютеру напрямую, без
внешнего USB-UART преобразователя (например, без микросхемы CP2102 или CH340). Он
выполняет две функции:
*   **Serial (CDC-ACM)**: Виртуальный COM-порт для вывода `Serial.print()` и получения данных.
*   **JTAG**: Интерфейс для отладки кода прямо через USB.

| Параметр | Значение |
| :--- | :--- |
| **Обозначение в коде (Arduino)** | `Serial` |
| **Пин D+** | GPIO 13 |
| **Пин D-** | GPIO 12 |
| **Тип** | USB 2.0 Full-Speed |
| **Особенности** | Позволяет программировать плату и смотреть отладку по одному кабелю USB-C . |

### 2. UART0 (UART0)
Это стандартный аппаратный UART, который обычно используется для вывода загрузочной информации (лог загрузчика) и может быть использован для связи с другими устройствами. На многих платах он дублирует вывод на тот же USB-разъем, что и USB Serial/JTAG, либо выведен на отдельные пины.

| Параметр | Значение |
| :--- | :--- |
| **Обозначение в коде (Arduino)** | `Serial0` |
| **TX (Передача)** | GPIO 16 |
| **RX (Прием)** | GPIO 17 |
| **Особенности** | По умолчанию используется для вывода логов (если не активирован USB CDC) . |

### 3. UART1 (UART1)
Дополнительный полнофункциональный UART, предназначенный для подключения внешних устройств (датчиков, GPS, GSM-модемов). Он поддерживает аппаратное управление потоком (RTS/CTS).

| Параметр | Значение |
| :--- | :--- |
| **Обозначение в коде (Arduino)** | `Serial1` |
| **TX (Передача)** | GPIO 7 |
| **RX (Прием)** | GPIO 6 |
| **RTS (запрос отправки)** | GPIO 4 |
| **CTS (готовность к отправке)** | GPIO 5 |
| **Особенности** | Идеален для подключения периферии. На некоторых платах (например, SparkFun) пины по умолчанию могут отличаться (4 и 5) . |

### Важные замечания

1.  **USB CDC On Boot (Важно для Arduino IDE)**
    При программировании плат ESP32-C6 в Arduino IDE вы найдете опцию меню **"USB CDC On Boot"**.
    *   Если включить её (**Enabled**), объект `Serial` будет указывать на **USB Serial/JTAG** (высокоскоростной порт через USB-C).
    *   Если отключить (**Disabled**), `Serial` будет указывать на аппаратный **UART0** (пины GPIO 16/17), а USB-порт станет доступен как `Serial0` (или не будет работать как COM-порт).
    *   `Serial1` всегда указывает на аппаратный UART1, независимо от этой настройки .

2.  **Гибкость назначения пинов (GPIO MUX)**
    В ESP32-C6 большинство периферийных устройств (включая UART) можно переназначить на любые другие свободные GPIO. Если стандартные пины (например, GPIO 16/17) заняты, вы можете в коде указать другие контакты для TX и RX.

3.  **Питание и уровни**
    Все UART порты (UART0, UART1) работают с логическими уровнями **3.3 В**. Подключая их к устройствам с 5V логикой (например, старым ардуино), необходимо использовать конвертер уровней.

### Сводная таблица

| Порт | Тип | Пин TX | Пин RX | Назначение |
| :--- | :--- | :--- | :--- | :--- |
| **USB Serial/JTAG** | USB | GPIO 13 (D+) | GPIO 12 (D-) | Программирование, отладка (JTAG), быстрая связь с ПК. |
| **UART0** | Аппаратный UART | GPIO 16 | GPIO 17 | Вывод логов загрузчика, общая цель. |
| **UART1** | Аппаратный UART | GPIO 7 | GPIO 6 | Подключение внешних устройств (GPS, датчики, RS485). |
