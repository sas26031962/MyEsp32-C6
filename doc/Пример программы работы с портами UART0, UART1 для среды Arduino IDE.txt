Ниже приведён пример программы для ESP32-C6 в среде Arduino IDE, демонстрирующий работу с аппаратными портами **UART0** (объект `Serial0`) и **UART1** (объект `Serial1`). Встроенный USB‑порт (`Serial`) используется для отладки и управления.

## Особенности ESP32-C6 в Arduino
- **`Serial`** — USB Serial/JTAG (виртуальный COM‑порт через USB‑C). Используется для программирования и общения с ПК.
- **`Serial0`** — аппаратный UART0 (по умолчанию TX = GPIO16, RX = GPIO17).
- **`Serial1`** — аппаратный UART1 (по умолчанию TX = GPIO7, RX = GPIO6).

Все аппаратные UART могут быть переназначены на любые другие GPIO с поддержкой ввода‑вывода.

## Пример кода

```cpp
void setup() {
  // Инициализация USB Serial (для отладки)
  Serial.begin(115200);
  delay(1000); // небольшая задержка для стабилизации USB
  Serial.println("ESP32-C6 UART example started");

  // Настройка UART0 на пользовательских пинах:
  // TX = GPIO16, RX = GPIO17, скорость 9600
  Serial0.begin(9600, SERIAL_8N1, 17, 16); // формат: скорость, конфигурация, RX пин, TX пин
  Serial0.println("UART0 initialized");

  // Настройка UART1 на пользовательских пинах:
  // TX = GPIO7, RX = GPIO6, скорость 115200
  Serial1.begin(115200, SERIAL_8N1, 6, 7);
  Serial1.println("UART1 initialized");
}

void loop() {
  // 1. Чтение из UART1 и вывод в USB Serial (монитор порта)
  if (Serial1.available()) {
    String data = Serial1.readStringUntil('\n');
    Serial.print("Received from UART1: ");
    Serial.println(data);
  }

  // 2. Чтение команды из USB Serial и отправка её в UART0
  if (Serial.available()) {
    String command = Serial.readStringUntil('\n');
    Serial0.print("Command: ");
    Serial0.println(command);
    Serial.println("Command sent to UART0");
  }

  delay(10); // небольшая задержка для стабильности
}
```

## Пояснения

1. **Инициализация с указанием пинов**  
   Метод `begin(baud, config, rxPin, txPin)` позволяет задать любые доступные GPIO для RX и TX.  
   В примере для `Serial0` использованы стандартные пины (16/17), для `Serial1` — тоже стандартные (7/6). Если плата имеет другую разводку, достаточно подставить нужные номера.

2. **Обмен данными**  
   - При получении данных по `Serial1` (например, от внешнего устройства) они выводятся в монитор порта через `Serial`.  
   - Ввод текста в мониторе порта и нажатие Enter отправляет строку в `Serial0`, что можно проверить, подключив к пинам 16/17 внешний USB‑UART преобразователь.

3. **Совместимость с загрузчиком**  
   UART0 по умолчанию используется для вывода логов загрузчика ESP-IDF. Если вы хотите использовать его исключительно для своих целей, отключите вывод логов в настройках платы (меню **Tools → Core Debug Level → None**) или переназначьте пины на другие.

4. **Настройка USB CDC On Boot**  
   В меню **Tools → USB CDC On Boot** выберите **Enabled**, чтобы `Serial` соответствовал USB‑порту. При этом `Serial0` и `Serial1` остаются аппаратными UART и работают независимо.

## Подключение внешних устройств

| Порт  | TX (передача) | RX (приём) | Рекомендуемое применение |
|-------|---------------|------------|--------------------------|
| UART0 | GPIO16        | GPIO17     | Обмен с датчиками, GPS    |
| UART1 | GPIO7         | GPIO6      | Подключение RS485, отладка |

**Внимание:** Все UART работают с логическими уровнями 3.3 В. Для связи с 5‑вольтовыми устройствами используйте согласователь уровней.

Этот пример можно легко адаптировать под любые сценарии двусторонней последовательной связи на ESP32-C6.
